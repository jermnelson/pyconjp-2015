About

This presentation is about the preliminary results for the development of a Linked Data Fragment Server - built using Python and Redis - for providing a lightweight mechanism to retrieve cached RDF triples for use by Semantic Web applications.

The Problem

Providing a full SPARQL endpoint for both small and large RDF linked-data sets is costly both in time and resources, especially for more complex queries.
